Snapie Auth API

A purpose-built Hive account gateway and signing proxy. Authenticates users via Google or email, creates custodial Hive accounts, and signs blockchain operations on their behalf.

Base URLhttps://auth.snapie.io/api
ProtocolJSON over HTTPS
AuthSession cookie (httpOnly) + CSRF token
Rate LimitsVaries per endpoint group

Overview

Snapie Auth bridges Web2 identity (Google, email) to Hive blockchain accounts. It handles the full lifecycle: account creation using pre-claimed tokens (ACTs), custodial key management, Hive operation signing, and voluntary key export (emancipation).

External services integrate via the Internal API using a shared Bearer token for account provisioning and sponsor token issuance.

Auth & Cookies

Sessions are stored in two cookies:

CookiePurposeFlags
snapieauth_sessionJWT RS256 session tokenhttpOnly, Secure, SameSite=Lax
snapieauth_csrfCSRF double-submit tokenSecure, SameSite=Lax (JS-readable)

All state-mutating requests must include the CSRF token as the x-csrf-token header. Read it from the snapieauth_csrf cookie with JavaScript.

Error Codes

{ "error": "error_code_string" }

Common codes: unauthorized, csrf_invalid, already_has_account, no_acts, invalid_username, emancipation_required, not_custodial.

GET /account/eligibility

GET/account/eligibilitySession

Check if the current user can create a free Hive account.

{
  "canCreate": true,
  "sponsored": true,

sponsored: true means a sponsor token was found for this email — the account creation fee is waived even for users who previously had an account.

POST /internal/issue-sponsor-token

Issue a one-time sponsor token for an email address. When the user registers with that email and creates a Hive account, the token is consumed and the account creation fee is waived (even for previously-account holders).
